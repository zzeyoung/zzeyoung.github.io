시각장애인을 위한 AI기반 폐플라스틱 분류 앱

캡스톤디자인

YOLOv11으로 PET병의 투명/유색을 실시간 분류하고, OCR로 라벨을 인식한 뒤 TTS 음성으로 분리배출 방법을 안내하는 iOS 앱

시연 영상 GitHub
역할 AI 모델 개발 및 앱 연동 팀 3인 (캡스톤디자인 2조) 기간 2025
문제 정의

2022년부터 투명/유색 플라스틱 분리배출이 의무화됐지만, 시각장애인은 물리적 구조와 색 구분이 어려워 가족이나 지인의 도움에 의존. 기존 앱은 분리수거 가이드나 대행 서비스 위주이고, 분리배출에 특화된 객체 탐지 + TTS 안내 시스템은 부재.

수행 내용
  • 26,000장 PET병 이미지 데이터셋 직접 구축 — 여러 종류의 PET병을 각기 다른 촬영 각도, 다양한 배경 환경에서 수집 · 라벨링
  • YOLOv11 모델 학습 — 투명 PET / 유색 PET 2클래스 분류, 하이퍼파라미터 튜닝으로 정확도 최적화
  • ByteTrack 객체 추적 도입 — 칼만 필터로 위치·속도 예측, IoU 기반 비용 행렬 생성, 헝가리안 알고리즘으로 최적 매칭, 트랙 상태 머신(tentative→confirmed→lost)으로 중복 탐지 제거
  • OCR로 PET병 라벨 텍스트 인식 → TTS 음성으로 분리배출 방법 안내하는 파이프라인 구현
  • Notion 캘린더 기반 일정 관리, 주차별 회의록·개발 문서 작성으로 팀 협업 체계화
트러블슈팅

바운딩 박스가 프레임마다 여러 개 잡히면서 TTS가 같은 객체를 반복 안내하는 문제 발생. ByteTrack을 도입해 객체별 고유 ID를 부여하고, IoU 임계값 0.3 미만인 객체는 무시하도록 설정해 안내 정확도를 개선.

쓱싹 — AI기반 디지털 탄소 절감 사진 정리 앱

taba 8기 프로젝트 우수상 🏆

AI가 사진의 메타데이터와 이미지를 분석해 중복·흐린 사진을 자동 분류하고, 삭제 시 절감된 탄소량을 나무 그루 수로 환산해 보여주는 앱. 만료 임박 기프티콘 OCR 알림까지 제공.

GitHub
역할 기획 주도 + 백엔드 + 프론트엔드(팀원 이탈 후 커버) 팀 3인 (taba 8기 1조)
수행 내용
  • 프로젝트 아이디어 제안 — 1인당 평균 900장 사진이 연간 10.6kg CO₂를 배출한다는 데이터를 근거로 기획
  • AI 사진 분석 파이프라인 설계 — 메타데이터(날짜, 위치, 파일명, 크기) + AI 분석(중복/유사 탐지, 흐림 감지, 객체 탐지, OCR)
  • 유사 사진 그룹화 알고리즘: pHash(이미지 회전 후 4방향 해시 비교), dHash(밝기 변화 기반), 히스토그램 chi-square 거리 — 3가지 중 2개 이상 유사하면 그룹화
  • 리마인드 기능 구현 — 6개월 지난 스크린샷, 만료 임박 기프티콘(바코드/QR OCR), 정보성 스크린샷 자동 알림
  • 환경 리포트 — 삭제 용량을 CO₂ 절감량과 나무 그루 수로 환산, 랭킹 시스템으로 동기 부여
  • 팀원 1명 이탈 후 프론트엔드를 직접 커버해 프로젝트 완수
  • Flutter(Dart)로 모바일 앱 프론트엔드 개발, Node.js + AWS EC2 백엔드, DynamoDB 데이터 저장, Firebase Auth 소셜 로그인(카카오톡/네이버) 구현
배운 점

팀원 이탈 시 개인별 Notion 기획 문서 작성 → 비교 논의 워크플로우를 제안해 갈등 해결. 감정이 아닌 문서 기반 합의가 팀 성과에 직결됨을 체감.

관광지가 나에게 말을 건다 — AI 관광 대화 서비스

2026 관광 프롬프톤

각 관광지에 MBTI 기반 고유 페르소나를 부여해, 관광지가 직접 사용자에게 말을 거는 AI 캐릭터 대화 서비스. 한국관광공사 OpenAPI를 실시간 연동해 혼잡도, 운영 정보, 오디오 가이드까지 캐릭터의 말투로 전달.

시연 영상
역할 팀장 · 프롬프트 설계 · UX 설계 · 웹 구현 팀 2인 (team 쓱싹) 기간 2026.06
수행 내용
  • 각 관광지에 성격, 말투, MBTI를 부여한 AI 캐릭터 페르소나 설계 — 경복궁은 INFJ 특유의 진중한 사극 말투, 양양 낙산해수욕장은 ENFP 서핑 분위기
  • 한국관광공사 OpenAPI 4종 연동 — 혼잡도 API(실시간 방문객 수를 캐릭터 컨디션으로 표현), 관광정보 API(역사·운영 정보), 오디오 가이드, 무장애 여행정보
  • GPS 방문 인증 & 캐릭터 도감 시스템 설계 — 미발견→발견→방문 3단계 수집, 카카오맵 핀으로 여행 기록 시각화
  • 서비스 전체 UX 플로우 설계 및 웹 프론트엔드 구현
배운 점

RAG 구현 과정에서 "기술은 멋있는 걸 쓰는 게 아니라 문제에 맞게 쓰는 거라는 걸 체감." 페르소나 설계부터 API 연동, 토큰 최적화까지 실제 서비스 레벨의 프롬프트 엔지니어링을 경험.

고령층 디지털 격차 데이터 분석

전공별AI활용(SW)

한국지능정보사회진흥원(NIA) 디지털정보격차실태조사 데이터를 분석하여 일반 국민 대비 고령층의 디지털 역량 격차를 시각화하고 정책 제안까지 수행

GitHub
역할 데이터 분석 전담 팀 2인 (3조) 데이터 7,000명 응답, 234개 변수
수행 내용
  • 공공데이터포털에서 NIA 디지털정보격차실태조사 통계자료(CSV) 수집
  • 코드북 기반으로 234개 변수 분석 → 분석 목적에 불필요한 133개 열 제거하여 101개로 축소 (ds.shape: 7000×234 → 7000×101)
  • 결측치가 많은 온라인 수강 항목은 별도 learning 데이터프레임으로 분리해 메모리 낭비 방지
  • 고령층/일반국민 그룹 분리 후 항목별 비교 — 모바일 기기 이용 능력, 검색·이메일·콘텐츠 서비스 이용률, 디지털 대전환 인식
  • matplotlib로 7개 이상의 비교 시각화 차트 제작
주요 분석 결과
  • 고령층 모바일 기기 이용 능력은 일반 국민의 절반 수준 — 특히 악성코드 검사(24.4% vs 70.3%), 문서 작성(35.9% vs 82.4%), 파일 이동(30.7% vs 83.4%) 격차가 심각
  • 교육콘텐츠 이용률은 고령층 10.9% vs 일반 국민 34.8%, 이메일은 16.4% vs 46.7%
  • 디지털 배움터 참여 고령층 중 90%가 "도움이 됐다"고 응답 → 배움터 홍보 강화 필요 제안
